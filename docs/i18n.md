# Internacionalização (i18n) — Supremacia Digital

Sistema de tradução próprio, leve, sem biblioteca externa. Idiomas: **pt** (canônico), **en**, **es**.

## Estrutura

```
client/src/i18n/
  index.ts              # translate(), interpolação {var}, fallback, detecção de idioma
  useI18n.ts            # store zustand (lang/setLang) + hook useT() + t() não-reativo
  locales/pt.json       # fonte canônica de chaves
  locales/en.json       # tipado contra pt — precisa de TODAS as chaves de pt
  locales/es.json       # idem
```

## Como usar

Em componentes React:

```tsx
import { useT } from '@/i18n/useI18n';

function Foo() {
  const t = useT();                       // reativo: re-renderiza ao trocar idioma
  return <button>{t('menu.newGame')}</button>;
}
```

Fora de componente (toasts, alerts, callbacks):

```ts
import { t } from '@/i18n/useI18n';       // lê o idioma atual do store
toast.error(t('error.badSave'));
```

Variáveis (interpolação `{nome}`):

```ts
t('hud.optionalActions', { used: 1, max: 3 })   // "Ações opcionais: 1/3"
t('bs.conqueredToast', { name: 'Brasil' })
```

Chave construída dinamicamente (precisa do cast):

```ts
import { TranslationKey } from '@/i18n';
t(`resource.${key}` as TranslationKey)           // resource.grain / .oil / .mineral
t(`phase.${stage}.name` as TranslationKey)
```

## Regras

1. **Nada hardcoded.** Todo texto novo de UI entra como chave nos três arquivos.
2. **`pt.json` define o conjunto de chaves.** Adicionou chave em pt? Adicione em en e es **na mesma hora** — `pt.json` é tipado e o `tsc` quebra se en/es estiverem incompletos.
3. **Fallback em cadeia:** idioma escolhido → pt → a própria chave (a tela nunca quebra com texto vazio).
4. **Convenção de chaves:** planas, pontuadas, por área — `menu.*`, `setup.*`, `phase.N.name`, `hud.*`, `bs.*` (BottomSheet), `combat.*`, `market.*`, `cards.*`, `prospect.*`, `sells.*`, `draw.*`, `present.*`, `lobby.*`, `tutorial.*`, `map.*`, `resource.*`, `common.*`. **Nomes de dados:** `terr.<id>`, `sea.<id>`, `faction.<id>`, `faction.<id>.short`, `company.<cardId>` (ver abaixo).
5. **Persistência:** o idioma escolhido fica em `localStorage('supremacia_lang')`; na 1ª visita é sugerido a partir de `navigator.language`.

## Nomes de dados (territórios, mares, facções, companhias)

Resolvidos pelo módulo [`i18n/names.ts`](../client/src/i18n/names.ts). O **id canônico**
continua sendo a fonte de verdade na lógica; os helpers só traduzem o RÓTULO exibido.

- **Hook reativo** `useNames()` → `{ territory, sea, location, faction, factionShort, company }`.
  Re-renderiza ao trocar de idioma. Use em componentes React.
- **Versões não-reativas** (`territoryName`, `seaName`, `locationName`, `factionName`,
  `factionShort`, `companyName`) leem o idioma atual do store — para o motor/callbacks.

Estado:

- **Territórios, mares e facções:** totalmente localizados (chaves `terr.<id>`, `sea.<id>`,
  `faction.<id>`, `faction.<id>.short`). A geografia real tem tradução estabelecida nos 3 idiomas.
- **Companhias:** os nomes em `resourceCards.ts` são marcas inventadas (criadas para evitar
  copyright). Por serem marcas, são **mantidas constantes entre idiomas por enquanto** —
  `company(cardId, fallback)` roteia pelo i18n com fallback para `card.companyName`. Localizar
  no futuro é só adicionar as chaves `company.<cardId>` nos 3 locales, **sem tocar em nenhum
  call site**.

## Pendências (follow-up, ainda em pt)

- **Mensagens dinâmicas do motor:** `eventLog` (NewsTicker / EventLogDrawer), `moveBlockMessage`,
  e os `title`/`description` dos passos da IA (`TurnPresentationPanel`). Geradas no `store.ts`/IA
  em pt. Os **números** dentro delas já respeitam o idioma; as **palavras** não. Localizar gerando
  por chave + variáveis (em vez de string pronta). Atenção: alguns componentes filtram o `eventLog`
  por prefixo de texto pt (ex.: `MarketDrawer` usa `e.message.startsWith('Vendeu')`) — ao localizar,
  trocar esses filtros por um campo estruturado (ex.: `event.kind`).

## Tutorial

O tutorial progressivo (`ui/TutorialCoach.tsx` + `stores/tutorialStore.ts`) lê todo o conteúdo do i18n (`tutorial.*`). Níveis: `full` (cards completos), `tips` (dica curta), `off`. Preferência em `localStorage('supremacia_tutorial_level')`; fases já vistas em `localStorage('supremacia_tutorial_seen')`. Cards automáticos aparecem na 1ª visita de cada fase nas primeiras `TUTORIAL_AUTO_ROUNDS` rodadas.

**Regra de manutenção:** toda feature/função criada, editada, corrigida ou removida obriga a revisar o tutorial — texto e gatilho — nos 3 idiomas, para que nunca descreva um comportamento que mudou. Tutorial desatualizado = bug.
